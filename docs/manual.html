SECTION 02

工作流详解

理解自动化任务的完整生命周期，从数据获取到 GitHub Pages 部署。

15:28 · 每日
任务计划触发 main.py
Windows 任务计划程序 "股票多周期K线图" 在每个交易日下午 15:30 触发，执行： D:\Anaconda3\python.exe E:\Myskills\stock-multi-period-big\main.py
Step 1 · 5秒
读取 stocks.txt
程序读取 stocks.txt，解析股票代码（支持 A股 / 港股 / 美股），过滤空行和注释行。
Step 2 · 30-60秒
从东方财富获取K线数据
通过东方财富 API 并行拉取 6 个周期数据：分时 / 日线 / 周线 / 月线 / 季线 / 年线。程序内置连接池、本地缓存和重试机制（3次指数退避），稳定性极高。
Step 3 · 60-120秒
生成多周期K线图 PNG
每只股票生成一张 2×3 拼图，包含 K线+均线、成交量、MACD 三大模块。采用黑色专业配色（#0d0d0d 背景），输出至 output/ 目录。
Step 4 · 30秒
生成 HTML 报告并推送 GitHub
调用 git_push.py：将最新 HTML 报告复制到 docs/index.html 作为固定入口，归档至 docs/archive/，最后 git add/commit/push。
Step 5 · 可选
推送通知（企业微信 / 飞书 / 邮件）
如已配置 config.yaml，自动发送卡片消息至企业微信/飞书群，或邮件。报告链接指向 stock-multi-period-big.pages.dev（Cloudflare Pages 国内加速）。
~15:35 · 完成
GitHub Pages 自动部署上线
GitHub 接收 push 后约 1 分钟内完成 Cloudflare Pages 部署，全网可访问。地址： stock-multi-period-big.pages.dev
SECTION 03

命令参考

所有可用的命令行参数，按场景分类。

基础运行

自动模式（读取 stocks.txt + 推送）
$python main.py
# 读取 stocks.txt，生成图，推送 GitHub Pages
本地预览（不推送到线上）
$python main.py --no-push
# 生成图+HTML，但不上传到 GitHub，适合调试
⚠️
禁止单独运行 python main.py --stocks XXX 不加 --no-push！这会把单只股票覆盖到 GitHub Pages 首页。请始终加上 --no-push 防止意外推送。

指定股票

单只股票
$python main.py --stocks 600036 --no-push
多只股票（空格分隔）
$python main.py --stocks 600036 000001 hk00700 AAPL --no-push

布局方案

方案A（默认）2×3 六周期
$python main.py --layout A
# 包含：分时/日/周/月/季/年
方案B 2×2 四周期
$python main.py --layout B
# 包含：日/周/月/季（去掉分时和年线）

通知渠道

企业微信 Webhook
$python main.py --wecom "https://qyapi.weixin.qq.com/cgi-bin/..."
飞书 Webhook
$python main.py --feishu "https://open.feishu.cn/..."
读取 config.yaml 发送所有配置的通知
$python main.py --notify
# 发送企业微信+飞书+邮件（从 config.yaml 读取 Webhook URL）
离线模式（仅使用本地缓存）
$python main.py --offline
# 不发网络请求，用上次成功的缓存数据，适合网络不稳定时调试
ℹ️
参数可以组合： python main.py --stocks 600036 --layout B --notify --no-push
以上命令：只生成 600036 的方案B图，发通知，但不上传 GitHub。

SECTION 06

通知渠道配置

支持企业微信、飞书、邮件三种推送方式，通过 config.yaml 统一配置。

💬

企业微信

通过 Webhook 机器人发送卡片消息，支持直接推送图片（<2MB），超过自动压缩。

🚀

飞书 Webhook

通过飞书群机器人 Webhook 发送卡片消息，携带报告链接。

📧

邮件 SMTP

支持 QQ 邮箱 SSL/STARTTLS 模式，发送完整 HTML 报告（图片 base64 内嵌）。

config.yaml 配置模板
📁 E:\Myskills\stock-multi-period-big\config.yaml.example
📄config.yaml
# 企业微信 Webhook（必填） wecom: webhook_url: "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY" # 飞书 Webhook（可选） feishu: webhook_url: "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_HOOK" # 邮件配置（可选） email: smtp_host: "smtp.qq.com" smtp_port: 465 use_ssl: true from_addr: "lzxs666@qq.com" to_addrs: - "lzxs666@qq.com" - "another@example.com" subject: "多周期K线图 {date}" # 授权码在 QQ 邮箱 → 设置 → 账户 → POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务 auth_code: "YOUR_AUTH_CODE"
⚠️
安全建议：将 config.yaml 加入 .gitignore，避免 Webhook URL 和邮箱授权码泄露到 GitHub。

SECTION 08

常见问题

遇到问题时先来这里找答案。

GitHub Pages 显示的是旧内容？
Cloudflare Pages 部署通常需要 1 分钟。可以在 stock-multi-period-big.pages.dev 按 Ctrl+F5 强制刷新。如仍有问题，检查 GitHub 仓库的 Actions 标签页是否有部署失败记录。
GitHub Pages 只显示一只股票？
很可能是之前手动运行 --stocks XXX 不带 --no-push 覆盖了 docs/index.html。解决方法：等下次自动化运行（15:30）自动恢复，或手动改回 stocks.txt 的完整列表后运行 python main.py。
网络不好，数据获取失败？
程序内置自动重试（最多 3 次，指数退避 1s/2s/4s）。如仍失败，自动使用上次成功的本地缓存数据（24h 有效）。也可加 --offline 参数强制离线模式。
企业微信图片发不出去？
企业微信限制图片 >2MB。程序会自动检测，当图片超过 2MB 时用 Pillow 压缩为 JPEG 后发送。如压缩后仍超过限制，会跳过图片只发文字链接。
想临时测试单只股票？
使用 python main.py --stocks 600036 --no-push，加上 --no-push 不会覆盖 GitHub Pages，方便本地调试。
需要添加新的股票？
直接在 stocks.txt 中添加一行（按格式：A股纯数字、港股 hk前缀、美股大写），保存即可。下次自动化运行会自动包含。
